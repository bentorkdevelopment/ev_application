# Skill: Test Case Generation

## Purpose

Generate meaningful test cases for features or code.

---

## When to Use

* After feature creation
* During validation step
* Before final delivery

---

## Responsibilities

1. Understand feature behavior
2. Identify inputs and outputs
3. Generate test scenarios

---

## Test Categories

### Functional Tests

* Normal expected behavior

### Edge Cases

* Empty inputs
* Boundary values
* Invalid data

### Failure Cases

* Error handling
* Exceptions

---

## Output Format

Each test case must include:

* Test Name
* Scenario
* Input
* Expected Result

---

## Guidelines

* Keep tests practical and realistic
* Avoid unnecessary complexity
* Cover important scenarios
